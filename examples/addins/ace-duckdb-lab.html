Engineering Feedback: How Geotab can Optimize This

To move beyond this proof-of-concept, Geotab Engineering should address the following:

  1. GCS CORS Configuration: Currently, the Google Cloud Storage bucket (planet-user-results-prod) does not allow GET requests from my.geotab.com origins. This forces us to download the entire file into a Blob first. If Geotab set the CORS origin to * or specific MyGeotab domains, DuckDB could stream the data directly using HTTP range requests, massively reducing browser memory usage.
  2. CSP Whitelisting: The default Content Security Policy for Add-in iframes is highly restrictive. Whitelisting connect-src storage.googleapis.com and common CDN domains would allow for native WASM extension loading (like the DuckDB JSON/Spatial extensions).
  3. Ace Response Schema: Standardizing the location of the signed_url across all Ace functions would eliminate the need for recursive object-crawling.
